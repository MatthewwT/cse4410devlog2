Olvidar Devlog 2

Abstract puzzle game designed by Mauricio, Antonio, Matthew, & Peter

Olvidar - The beginning


After the introduction of the tutorial, we added the first room that the player will experience. This is where we added our first puzzle which was designed by one of our team members. Each consecutive puzzle will be deisgned by each team member, and the puzzle ideas we have so far are as shown below.


The first puzzle consists of three pictures of a 3x3 grid where each grid will be filled with a different color. The player will then have an empty grid in front of them and have to figure out how to complete the puzzle in order to unlock the next area or the next puzzle in the room.

The second puzzle will deal with the periodic table. The player will have to figure out using the elements and numbers of the periodic table to get a key word and type it into the popup prompt.

The next puzzle (no image so far) will deal with NPC interaction and vials. The player will have to bring a vial to the NPC, where the NPC will respond with a prompt based on the vials smell. The player will have to place the vials in the correct sequence based on picture hints. For example, if the vial smells like flowers, the player will have to place it in front of a picture with flowers.

Member's progress - 2


Movement - Matthew

In this update, we have fixed the player sticking to walls. The issue with this was a checkbox in unity for the physics which made the player able to stick to walls.

The future update will still need to implement a character sprite in order to begin character animations.


Scene Transition - Mauricio

The updated scene transition is still a work in progress, but the vector work is coming along smoothly.

The issue was that the player model was not spawning correctly due to not having the vectors moving the player model.

A study into why this was happening is in place and will be fully implemented before the demo of the live game.


Interactions - Peter/Antonio

After going through some Unity tutorials for 2D platformers, we have a better idea on how to implement interactions with NPC's and objects.

One approach is to use a box collider in Unity and code it to whenever the character touches the NPC, the NPC will give them a prompt which will close whenever the player walks away. This apporach is easy to execute but is not esthetically pleasing.

Another approach we came up with was to have the NPC as a background interactable object in which the player can use their mouse to click on the NPC or have a popup dialog which the player can then press a button to interact with the NPC.
